Searching Algorithms

Definition

Linear Search finds a target value in an array by checking each element sequentially from left to right.

Pre-condition

  • None. Works on sorted, unsorted, single, or empty collections.

How it Works

  • Start from index 0
  • Compare current element with target
  • If equal, target found at current index
  • If not equal, move to the next index
  • If end of array is reached, target is not found

Common Mistakes

  • Returning target not found inside the loop before checking all elements.
  • Iterating past the array bounds (out of bounds error).
  • Not handling duplicate values correctly.

Where to Use

Use it for small unsorted arrays, linked lists, or simple lookups where the overhead of sorting is higher.

    When to Choose Which Sort?

    • Bubble Sort: Best for learning and very small arrays.
    • Selection Sort: Useful when writes/swaps are expensive.
    • Insertion Sort: Great for nearly sorted arrays and online insertion.

    Real World Examples

    • Insertion Sort appears when arranging cards in your hand.
    • Selection-like logic appears when repeatedly picking the minimum-priority task.
    • Bubble Sort is great for understanding adjacent swaps in local optimization.
